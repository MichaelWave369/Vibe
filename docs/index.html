What is Vibe?

Vibe is an intent-first programming language and compiler prototype. It treats intent as first-class source material and verifies preservation before code emission.

Intention is source code.

Preservation is compilation truth.

Founding Law

Vibe compiles only when implementation preserves meaning above the bridge threshold.

Current emit targets: python, typescript.

epsilon_floor = 0.02
measurement_safe_ratio = 0.85

compile fails if epsilon_post <= epsilon_floor
compile fails if measurement_ratio < measurement_safe_ratio

measurement_ratio = epsilon_post / max(epsilon_pre, epsilon_floor)

Compiler Flow

  1. .vibe source
  2. parse
  3. AST/IR normalization
  4. validate + incremental cache compare
  5. generate + normalize obligations
  6. evaluate obligations via verification backend
  7. emit code (only on pass)
  8. report metrics + verdict

Bridge Verification

Verifier outputs qixel metrics, alignment terms, bridge score, obligation summaries, backend metadata, and a preservation verdict.

Default backend: heuristic. Architecture includes placeholder symbolic plus a first real smt subset backend for founding-law bridge checks and simple scalar/boolean predicates.

Vibe also reports structural intent/emission correspondence with drift categories: matched, partial, missing, extra, and unknown (structural layer only; not full behavioral equivalence).

Epsilon estimation can be empirically calibrated from a local corpus via vibec calibrate, while preserving founding-law safety gates.

Verification can also emit deterministic preservation proof artifacts (.vibe.proof.json) for transportable inspection via inspect-proof.

Compile/verify can synthesize multiple deterministic candidates and emit only the highest-ranked passing candidate under bridge-gated verification.

Optional intent-guided test generation is available with --with-tests: compile emits deterministic test files next to emitted code, while report/proof surfaces preserve-rule and constraint coverage metadata with explicit uncovered/partial honesty.

Phase 3.3 adds deterministic bridge-gated refinement (--refine, --max-iters): failed rounds feed structured counterexample signals into the next synthesis round, and only a passing refined candidate may emit.

Phase 3.4 adds semantic intent diff (vibec diff old.vibe new.vibe) over normalized intent structures, so Vibe reports how programmer intent changed (goal/inputs/outputs/preserve/constraints/ bridge/emit/Tesla/Agentora/AgentCeption), distinct from emitted-code drift.

Phase 4.1 adds semantic types: intent values now carry meaning qualifiers (e.g. deterministic, secret-sensitive, bridge-critical) derived from preserve/constraint/bridge semantics and surfaced in IR/report diagnostics without replacing preservation obligations.

Phase 4.2 adds effect types: Vibe now tracks inferred behavioral effects (pure/io/stateful/ nondeterministic/network/fallback/bridge-critical) and reports effect mismatches as compile-time obligations while keeping unknown effects visible.

Phase 4.3 adds resource types: Vibe reports inferred resource profiles (memory/cpu/latency/allocation/ iteration hints) and surfaces resource mismatches as preservation obligations.

Phase 4.4 adds intent-seeded type inference: declared intent IO + preserve/constraint/bridge/declaration surfaces seed deterministic inferred bindings, unresolved points, and inference-derived obligations. Inference remains partial and explicit; preservation law still governs emission truth.

Phase 5.1 adds first-class agent graphs: native agent and orchestrate syntax with typed receives/emits boundaries, per-agent contracts, edge/fallback validation, and graph-derived obligations surfaced in IR/report/proof while runtime orchestration semantics remain intentionally partial.

Phase 5.2 adds bridge propagation across agent boundaries: every orchestration edge now receives deterministic compatibility checks + an edge bridge score, with a propagated pipeline score (product(edge_scores)) and boundary-critical failures that can block compile.

Phase 5.3 adds AgentCeption delegation analysis: native delegate Parent -> Child declarations, static proof inheritance for preserve/constraint/bridge contracts, contract-weakening detection, and recursion risk surfacing when delegation chains lack explicit depth/stop reasoning.

Phase 5.4 adds runtime monitor metadata + event evaluation: emitted monitor configs track agents/edges/ thresholds, and offline runtime events are scored for drift/alerts/fallback pressure relative to compiled contracts (observational only; compile-time proof remains authoritative).

Phase 6.2 adds a local-first intent registry: publish, search, registry-inspect, and compat now index reusable package intent contracts with proof/build metadata in deterministic local JSON entries. This is local publication only; a hosted registry remains a future roadmap phase.
