What is Vibe?

Vibe is an intent-first language and compiler. Pipeline: .vibe -> parse -> typed IR -> verify -> proof/report -> emit.

Compile fails when preservation gates fail.

Reports/proofs are deterministic and inspectable.

Local-first workflows: package manager, registry, LSP, CI bridge checks.

Quickstart

python -m pip install -e .
vibec verify vibe/examples/payment_router.vibe
vibec compile vibe/examples/payment_router.vibe
vibec verify-proof vibe/examples/payment_router.vibe
python -m pytest -q

Current status

Implemented slices include parser/IR foundations, verification + proof artifacts, local package manager, local registry, LSP, CI bridge-check integration, cross-domain profiles (hardware/scientific/legal/genomics), and Phase 8.x meta slices for bounded self-hosting, semantic versioning from intent diff, intent negotiation, and an initial standard library.

Standard Library (Phase 8.4)

Vibe now includes an initial bounded standard library of reusable intent-contract packages: vibe_http, vibe_payment, vibe_vector, and vibe_agent.

These packages are real local packages (vibe.toml + modules) that flow through manifest-check, build, verify-proof, and local registry publish/search/inspect operations.

Scope note: this is an initial package set, not complete standard-library coverage.

Verification and proof scope

Vibe emits machine-checkable proof metadata in deterministic .vibe.proof.json artifacts. This is a real verification pipeline, but not a claim of complete formal verification for all program behavior.

Roadmap

  • Expand standard-library package coverage and inter-package contracts.
  • Deepen semver and compatibility rule coverage.
  • Extend intent negotiation for richer multi-party contracts.
  • Expand bounded self-hosting slices toward fuller compiler coverage.
